## Part C — NTP Timezone Verification

The NTP feature uses `sntp_simple()` which returns seconds since the Unix epoch (UTC).
`sys_clock_settime(SYS_CLOCK_REALTIME, ...)` stores this value in the Zephyr realtime clock.
`CONFIG_LOG_TIMESTAMP_USE_REALTIME=y` formats log timestamps directly from the realtime clock.

**Verification:** epoch 1778787292 → `2026-05-14 19:34:52 UTC` (confirmed by Python `datetime.fromtimestamp(1778787292, tz=timezone.utc)`). The displayed log timestamp `[2026-05-14 19:34:52.299,000]` matches exactly.

**Conclusion:** All timestamps are **UTC**. No timezone conversion is applied. This is the expected and correct behavior for an embedded device without timezone configuration.
